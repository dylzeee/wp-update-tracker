WP Update Tracker is a WordPress plugin that tracks updates to the WordPress core, plugins, themes, and settings changes.

== Description ==

WP Update Tracker is a robust tool for administrators who want to keep a detailed record of all updates made to their WordPress site. The plugin tracks updates to the WordPress core, plugins, and themes, as well as settings changes, and provides a modern interface to view, filter, and export these logs.

**Features:**

- Tracks updates for:
  - WordPress Core
  - Plugins
  - Themes
  - Settings Changes
- Advanced Log Viewer:
  - Filter by event type and date range.
  - Export logs as CSV.
  - Modern, paginated table for easy navigation.
- User-friendly settings page to enable or disable specific tracking.

== Installation ==

1. Download the plugin as a `.zip` file or install directly from the WordPress Plugin Directory.
2. Navigate to **Plugins > Add New** in your WordPress dashboard.
3. Click **Upload Plugin**, select the downloaded file, and click **Install Now**.
4. Activate the plugin via the Plugins menu.

== Usage ==

1. After activating the plugin, navigate to **Update Tracker > Settings** to configure:
   - Enable or disable tracking for core updates, plugin updates, theme updates, and settings changes.
2. Go to **Update Tracker > Log Viewer** to view logs:
   - Use filters to narrow down logs by event type and date range.
   - Export logs as CSV for offline analysis.

== Frequently Asked Questions (FAQ) ==

= What updates does this plugin track? =  
The plugin tracks updates to the WordPress core, plugins, themes, and changes made to settings.

= Can I disable tracking for specific update types? =  
Yes, you can enable or disable tracking for core, plugin, theme, or settings updates via the **Update Tracker > Settings** page.

= How do I export logs? =  
Navigate to **Update Tracker > Log Viewer**, apply your desired filters, and click the **Export as CSV** button.

= Is this plugin resource-intensive? =  
The plugin is designed to be lightweight, using optimized database queries to store and retrieve logs.
